# C-7 設計案: 連続重み粒子フィルタ（ESSリサンプリング＋窓付き若返り）

Status: reviewed（2026-07-17 codex + cursor の2系統レビュー済み・指摘反映済み。
レビュー要点は末尾）

## 背景と狙い

v7 キャンペーンの結論（tuning/README.md 末尾）:

- 機構追加も SPSA 第3・4ラウンドも vs estimator_v6 で全て中立/不発
- オラクル測定で**反則経済に 36pt の伸びしろが実在**（全反則回避 86.2%±4.8 /
  王手中のみ回避 59.5%±7.1）。スカラー係数の調整では届かない
- kakunari シナリオで**粒子フィルタの中盤死**が判明: 実戦的な戦術連鎖
  （69角打の王手 → 同飛 → 同桂成で取得駒=飛車、を全て満たす粒子が 1137 個中 0）で
  全滅し、初期局面からの前向きリプレイは予算10倍でも復活 0。以後 12 手番ブラインドで
  迷走（正解の王手 0/20、指し継ぎ勝率 4/10）

### C-7 の位置づけ（cursor レビューで格下げ・分解）

当初は「信念の崩壊 → p_legal 崩壊 → 反則 → 36pt」と一本の因果で書いていたが、
レビュー指摘によりこれは過大主張。オラクルが与える価値は
①反則予算を減らさない ②`foul_tried` による無料の仮説消去 ③合法手に当たるまでの
無料再試行、の3つで、C-7 が直接回収できるのは①の一部（p_legal の質）だけ。
また check_nofoul(59.5%) と nofoul(86.2%) の差は王手中以外の反則
（経路封鎖・打ちマス等）で、CheckSolver の外側の話も多い。

よって C-7 の健全な位置づけは:

- **主目的: 中盤死という構造的バグの修正と、信念の下限保証**
  （kakunari で実証された全滅→ブラインド迷走を無くす）
- **反則経済 36pt は「期待しないが事後に測る」副次指標**。
  analyze の反則内訳（王手解消失敗/飛び込み/経路封鎖/打ちマス）別の増減で測る

評価式は `combine_score = (p_legal·gain).min(gain) − (1−p_legal)·foul_cost` で、
p_legal の入力は粒子集合そのもの。二値整合フィルタの構造的限界（全滅か等重みか）を
直すのが C-7 であり、「合法性知識 → 着手価値」変換の入力品質を上げる構造改修。

## 現状の構造（変更前の要約）

- 制約適用は**二値棄却**。例外は (a) ソフト救済: 厳密生存 < target/4 のとき情報系制約
  （王手宣言・反則説明）だけ緩和して penalty+1（評価側 0.5^penalty）、物理系
  （自手合法性・取得駒種・取られたマス）は緩和しない。(b) SIR logw: 相手手の
  整合クラス質量比 r = class_mass/total_mass を対数累積（keima 対応で導入済み）
- 補充（replenish）: penalty 昇順ソート → target へ truncate → 不足は複製。
  枯渇時は**初期局面から**の前向きリプレイ＋限定バックトラック（replay_once、
  step_budget = n×4+32）
- 評価側（strategy.rs stratified_sample）: 指紋ユニーク化 → soft_decay^penalty ×
  exp(θ·φ)（likelihood.rs、平均1正規化）× exp(logw − max) → 玉位置で層化抽出

## 設計の柱

### D1: 重み表現の統一 — penalty を連続 logw へ吸収

粒子状態を `(Position, logw: f64)` に一本化する。penalties ベクタは廃止。

- 相手手の r 累積（既存 SIR）はそのまま logw へ
- 情報系の不一致（王手宣言・反則の説明失敗）は棄却でなく `logw += ln(ε_info)`
  で生かす（初期値 0.1、SPSA 対象）。**適用は現行どおり人口ゲート付き**
  （厳密生存 < target/4 のときだけ発動。cursor 指摘: ゲートを外して常時適用に
  すると、王手宣言のたびに説明失敗粒子が大量に残って ESS が毎手閾値割れ →
  リサンプリング頻発 → 多様性崩壊、かつ「厳密が足りるときは情報不一致を棄却」
  という現行較正が消えて P1 の不変条件①が定義不能になる）。
  つまり D1 は「ゲートは維持し、減衰だけ 0.5^penalty → ε_info^info_miss へ
  連続化」する変更
- **info_miss カウンタは logw と別に保持する**（レビュー指摘: 重みに吸収すると
  「ソフトは独立証拠ではない」という較正が消える）。上限（現 PENALTY_CAP=3 相当）
  超過で棄却する点は維持（近似の信用限界）。D2 のリサンプリングでも**リセットしない**
- 物理系の不一致は引き続き即時緩和しない（D3 の修復が先。最後の砦は D4）

互換性: 評価側 stratified_sample のインターフェースは
`(particles, log_weights, info_miss, phys_taint)` になる。soft_decay は ε_info に統合。

### D2: ESS ベースの systematic resampling

replenish の「ソート → truncate → 複製」を標準的な SIR に置き換える。

- 正規化重み ŵᵢ ∝ exp(logwᵢ − max) で **ESS = 1/Σŵᵢ²** を毎 update 監視
- `ESS < target × ess_threshold`（初期値 0.5）で **systematic resampling**
  （低分散・O(n)）→ 全粒子 logw = 0 にリセット。**info_miss / phys_taint は
  リセットしない**（履歴の信用情報は重みと別勘定）
- リサンプリングしないターンは logw を保持し評価側がそのまま使う（現行と同じ）

評価側との整合で重要な3点:

1. **リサンプリング後の質量を評価側ユニーク化で捨てない**（レビュー最重要指摘）:
   リサンプリング後は「複製数そのものが事後質量」だが、現行 stratified_sample の
   指紋ユニーク化は複製を1個に潰すため、D2 の効果が評価時に消える。
   対策: ユニーク化の際に**同一指紋の multiplicity を重みへ畳み込む**
   （unique の重み = Σ 個体重み。現行の「最良個体で代表」から変更）。
   これを検証する不変条件テストを P1 の合格条件に含める。
   **較正アンカーの再定義が前提**（cursor 指摘）: 現行 legacy_mass は
   「penalty 昇順の先頭 min(eval, unique) 件の soft 重み和」だが、Σ重み方式では
   unique 数が減り各行のベース重みが増えるため `take(eval_particles)` の意味が
   変わる。P1 の不変条件①（旧 p_legal 一致）を書く**前に** multiplicity 導入後の
   legacy_mass 定義を確定する: 新定義 = 「multiplicity 畳み込み前の**個体単位**で
   penalty 昇順の先頭 min(eval, 個体数) 件の soft 重み和」（= 旧方式と同じ個体
   集合で計る）。これなら旧世界との一致テストが well-defined になる
2. **二重カウント回避**: リサンプリング後は logw=0 なので、評価側の
   exp(logw) 乗算は自然に無効化される。リセットを跨いだ二重適用は起きない
   （keima の r 重みが二重適用されないことをテストで固定する）
3. **健全度（confidence）は ESS 単独では作らない**（レビュー指摘: リサンプリングで
   ESS は回復するが情報は増えない）。フィルタがエクスポートする指標:
   - `strict_unique_effective_n` = 厳密整合（info_miss=0, phys_taint=0）粒子の
     ユニーク実効数 → **p_legal の事前ブレンドはこれで計る**（現行 degen の後継）
   - ユニーク指紋数、strict / info-soft / phys-taint の質量比
   - blind_turns（全滅・強制救済ターン数）と直近の若返り成功率
   攻め項ゲートの confidence はこれらの合成（最小実装: strict 質量比 ×
   decay^blind_turns）

### D3: 窓付き若返り（rejuvenation）— 中盤死対策の本丸

kakunari の教訓は「初期局面からの前向きリプレイは手数に対して指数的に失敗する」。
若返りは**生きている（または直前まで生きていた）粒子から直近だけ引き直す**。

- 各粒子に**直近の相手決定点のスナップショット**をリングバッファで持つ
  （窓は adaptive: まず k=2、失敗したら 4、8 と広げる。レビュー指摘採用 —
  固定 k だと「原因が k+1 手前」を拾えず、常時大きい k はコスト過剰。
  メモリは Position × target × k_max ≈ 数MB で許容）
- 制約適用で物理棄却された粒子は、即死でなく**巻き戻して相手手を引き直す**。
  **初期実装は「直近 j 手の相手決定点へ一様に戻る」**（現行バックトラックと同じ
  規則。cursor 指摘: 「関係する決定点を優先」ヒューリスティックは、角打→同飛→
  同桂成のような連鎖では真因が着地マス/駒種一致より前の配置手にあることが多く、
  弱い実装を置くと adaptive k を広げても間違った分岐を引き直すだけになる。
  依存解析ベースの優先順位付けは**別PR**とし、P2 の合格条件は一様巻き戻し前提で
  書く）。現 replay_once のバックトラックと同じ機構だが、開始点が初期局面でなく
  窓内。1回の修復コストが総手数に依存しない
- **制約後読みガイド**: 巻き戻した決定点で相手手をサンプルするとき、これから
  再適用する制約列を提案分布のマスク／ブーストに使う。ガイドの表現は
  「利きを作る」ではなく**状態条件**で書く（レビュー指摘採用 — 利きだけでは
  説明にならないケースがある）:
  - 後続の `MyMove(to=X, captured=R)` → その時刻の直前に **X に相手の R が存在する**
    こと。これを満たしうる手（R を X へ運ぶ／X に打つ）をブースト（×guide_boost、
    初期値 8.0）
  - 後続の `OpponentMoved(captured_at=X)` → その相手手の着地が X であること
  - 後続の王手宣言（の有無）→ 自玉への利き状態の一致
- **importance 補正は窓全体のパス比で払い、既存の r 会計と代数的に接続する**
  （codex+cursor 指摘の合成。ここが実装前に閉じるべき最大の穴だった）。会計規約:
  1. **スナップショットは `(Position, logw, info_miss)` を持つ**。巻き戻し時は
     logw もスナップショット値へ**巻き戻す**（= 棄却までに払った窓内の旧 r は
     全部捨てる。「差分だけ足す」と旧 r が残って二重になる）
  2. 引き直した各決定点では、既存と同一の生成モデルで
     `Δlogw = ln(r) + ln( p_class(chosen) / g_class(chosen) )` を累積する。
     r = 整合クラスの素の事前質量 / 全合法手の素の事前質量（既存どおり）、
     p_class / g_class = 整合クラス内での素の事前分布／ガイド付き提案分布に
     おける選択手の確率。これで修復粒子と非修復生存粒子の logw が**同じ生成
     モデルの絶対規約**に載り、keima の r 相対比較が窓の内外で壊れない
  3. **ガイドの support 条件**: マスク（確率0化）は「後続制約に必ず失敗すると
     証明できる経路」だけに許す。素の提案で正の確率を持ち成功しうる経路を
     マスクすると p_class/g_class が定義できなくなる。それ以外はブーストに
     留める（g_class の正規化に含めて正直に払う）
  4. 実装前に**「窓長2・1粒子」の数式ウォークスルー**をテストコード
     （または本文書の付録）として固定し、(a) 巻き戻し→引き直しで二重計上が
     ないこと (b) ガイドあり/なしで重み付き分布が一致することを検証する
- 若返りが全滅した場合のみ、従来の初期局面リプレイ（これにも同じ制約後読み
  ガイドを適用して打率を上げる）へフォールバック

これは resample-move（Gilks & Berzuini）に着想を得た SMC の局所再提案
（厳密な MCMC 不変分布は持たない近似。だからこそ上記のパス比補正を正直に払う）。
リサンプリング直後の複製粒子の多様性回復にも同じカーネルを使える
（複製粒子の直近数手を引き直す）。

公平性の境界: ガイドが参照する「後続制約」はすべて**観測済み**イベント
（observation.rs の全量）であり、未来情報ではない。コード上も ObservationLog
由来の制約列だけを入力に取る型にして境界を明示する。

### D4: 最後の砦 — ε_phys（完全ブラインドの回避、用途を限定）

D3 でも修復できない場合に限り、物理不整合の粒子を `logw += ln(ε_phys)`
（初期値 0.01）＋ `phys_taint` カウンタ +1（上限 1〜2）で残す。

レビュー指摘を受けた**毒化対策**（このまま通常評価に混ぜない）:

- `phys_taint` は logw と**別に永続化**し、D2 のリサンプリングでもリセットしない
  （リセットすると嘘粒子が通常粒子へ昇格する — レビュー指摘の失敗モード）
- **用途は「玉位置分布のみ」に限定**（cursor 指摘で当初案からさらに絞った:
  王周辺圧力・攻め項は粒子の玉位置と利きそのものであり嘘粒子の毒が最も出やすい
  ため大域項からも除外する）。phys_taint 粒子が寄与してよいのは玉位置仮説の
  タイブレーク程度で、**p_legal・捕獲期待値・取られリスク・圧力/攻め・
  層化の層キーからは全て除外**。評価側は strict / info-soft / phys-taint を
  区別できる形で受け取る（D1 のインターフェース）
- ε_phys 粒子が混ざるターンは blind_turns にカウントし、confidence を下げて
  事前ブレンドと CheckSolver（粒子非依存の王手制約推論）側へ重心を移す
- **evaluate 側のリファクタコストに注意**: 現行 evaluate は `(&Position, f64)` の
  単一ループで penalty すら受け取らない。用途分離にはサンプルの2系統化か
  `(pos, w_local, w_global, flags)` へのループ分割が必要で、改修コストが大きい。
  → **P3 は任意フェーズ**とし、P2 の物理修復だけで全滅ゼロに近づくなら
  ε_phys 自体を見送る（先に「D3 だけで足りるか」を検証する）
- kakunari のユーザー指摘（テンポ収支・負の証拠・利き枚数分布）のような
  高次の信念は本設計のスコープ外（粒子が生きてさえいれば θ·φ／r で部分的に
  表現される、が前提）。ansatsu（観測ゼロの忍び込み）も別課題として据え置き

### D5（任意・後段）: 観測尤度 θ·φ をリサンプリング時のポテンシャルに

現在 exp(θ·φ) は評価時のみ。リサンプリング重みへ `λ·(θ·φ)` を混ぜると
「真の局面に近い粒子」が生き残りやすくなる。ただし評価側の乗算と二重に
なるため、**採るなら評価側の適用をフィルタ側へ移す**（どちらか一方）。
粒子尤度単体はアリーナ中立だった実績（v7）から優先度は低く、P4 の SPSA で
λ=0 を含めて探索する。

## 実装フェーズと合格条件

各フェーズごとに独立にマージ判定する。**測定はシナリオスイート＋
match_seed ペアのアリーナ**（アリーナ単独は対人挙動の改善を検出できない）。

P1 は「非劣化の証明」ではなく **D3 の土台**と割り切る（レビュー指摘: 200局では
非劣化を証明できないし、D2 単体はリサンプリングで多様性を減らすため中盤死を
むしろ悪化させうる）。P1 の主武器はアリーナでなく決定論的な不変条件テスト。

| フェーズ | 内容 | 合格条件 |
|---|---|---|
| P1 | D1+D2（penalty→logw、multiplicity 保持のユニーク化、ESS リサンプリング、健全度指標の輸出） | **不変条件テスト**: ①全 logw=0・ソフトなしで旧 p_legal と一致（multiplicity 導入後の legacy_mass 新定義で） ②リサンプリング後の multiplicity が評価重みへ反映 ③リサンプリング前後で統計量（玉位置分布等）の重み付き期待値が**許容誤差つき・多シード平均で**保存（cursor 指摘: systematic resampling は期待値の意味でのみ不偏。パスワイズの決定論 assert はフレークする） ④ε_info 適用後の info_miss 上限と confidence が期待通り ⑤keima の r 重みがリサンプリングを跨いで二重適用されない。＋**シナリオ回帰は keima のみ**（プローブ 100% 維持。kakunari は物理全滅なので D1+D2 では改善せず、リサンプリングの多様性低下でむしろ悪化しうる — 現状値維持を課すと P1 で詰まる。kakunari は P2 の主指標へ）。アリーナは参考値（大負けしていないことの確認のみ） |

## 実装フェーズでの設計外判断（2026-07-18、codex 実装レビュー済み）

P1/P2 の実装中に設計文書に無い判断を3つ行い、codex の実装レビュー
（`git diff main...HEAD` + 作業ツリー）に掛けた。

1. **rebase_cidx（logw スケールの再ベース点）**: 実装中に P1 の会計バグを発見 —
   リサンプリングで logw を 0 にリセットした後、初期リプレイの新粒子が
   絶対スケール（全制約の累積 r）を持つと、生存粒子に対して常に不当に軽くなる。
   対策として「リサンプリング時に rebase_cidx = 現制約数 を記録し、以後の
   リプレイ粒子は rebase 以降の累積だけ課金する」（それ以前の質量は集団の
   典型と同じとみなす近似。ソフト救済の strict_dlw_median と同じ哲学）。
   **codex の注意**: この近似は不偏ではない — 過去観測で低尤度だった歴史を
   条件付きリプレイが過大注入しうる。P3/P4 で「rebase 後に追加された
   リプレイ粒子の質量比」の診断か、補充粒子の総質量 cap を検討する
2. **墓場（graveyard）**: 厳密全滅時に棄却粒子（スナップショット窓つき）を
   最大128個保管し、以後のターンで復活を試みる（kakunari の「一度全滅すると
   hist ごと消えて永久にブラインド」対策）。物理的にはスナップショット時点まで
   整合していた歴史なので嘘粒子ではない（info_miss>0 の由来粒子は情報観測に
   miss 分汚染されるが、miss は復活後も維持されるので較正は保たれる）。
   復活粒子の logw は、ソフトが生きていれば snap.logw から（スケール継続）、
   完全全滅後は rebase 規約（0起点）で揃える。stale（決定点から24制約超）で破棄
3. **失敗ヒストグラム（TSUITATE_FILTER_DEBUG=1）**: リプレイ・若返りが失敗した
   制約 index の分布を収集。kakunari の needle が c25（46–48手目の
   桂打込→角打王手→同飛→同桂成の連鎖）に集中していることを特定した計測手段

codex 実装レビューの修正指摘（反映済み）:

- 【最重要】墓場復活で返す hist の旧スナップショットが旧集団スケールのまま →

  **厳密生存数**で判定する（ソフトは独立証拠ではない — 設計どおりに戻した）

## P3 追補: ブラインド玉攻め勾配（2026-07-19、採用）

kakunari の指し継ぎに 90手足切り（`*scenario limit=90`）を導入して измерした
ところ、P3 時点の遂行は 2/20（初手の6割が無目的手）。一方 diag に追加した
「taint 込み全粒子の玉位置分布 vs 真実」で、**taint 粒子の玉位置信念は
91.8% で真実（4g）に集中**していることが判明 — 問題は信念でなく、評価が
それを使えないこと（taint の用途を王手ソルバー投票に限定していたため、
攻め側の評価は事前分布のみで動いていた）。

対応: クリーン粒子全滅時に限り、taint 粒子から**玉位置分布だけ**を抽出し
（深度減衰・上限は投票と同じガード）、「自駒だけの盤面で着地駒が信念玉マスへ
利きを作る手」へ BLIND_KING_ATTACK_W×信念確率のボーナスを足す。
盤面の嘘（幽霊取り等）は評価に一切入れない。

結果: **指し継ぎ 2/20 → 13/20（65%）**、勝ち局の追加手数 中央値 17→5
（最短1手）、初手は 5h4h+（真実の玉への直接王手）16/20。
信念→着手価値の変換の欠落が1点直るだけで遂行が跳ねる実例。
